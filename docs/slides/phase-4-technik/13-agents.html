Ein Agent ist kein Synonym für „gute AI". Er ist eine Orchestrierung aus Modell, Werkzeugen, Kontext und Kontrolllogik.

Function Calling

Über Function Calling fordert ein Modell einen strukturierten Tool-Aufruf an. Die Anwendung führt die Funktion aus und sendet das Ergebnis zurück.

  • Das Modell erhält dadurch keine direkten Datenbankrechte.
  • Kann nicht eigenmächtig beliebige Befehle ausführen.
  • Pro Tool: Zweck, typisierte Eingaben, Ergebnisformat dokumentieren.
  • Schreibende oder irreversible Tools brauchen Freigaben.

Die Anwendung muss maximale Schleifenanzahl, erlaubte Tools und Zeit pro Schritt technisch durchsetzen — ein Prompt reicht nicht.

Speicher, Kontext und Planung

StufeInhaltLebensdauer
Kurzzeitkontextaktuelle Anfrage, offene Arbeitsschritteeine Konversation / ein Lauf
ArbeitsgedächtnisZwischenstände, Beobachtungen, offene Punkteein Agentenlauf
Langzeitspeicherbewusst ausgewählte, überprüfte Informationenmehrere Läufe, mit Löschregeln

Sensible Daten, Zugangsdaten und Modell-Annahmen gehören nicht in einen persistenten Speicher.

Beim Plan wird ein Ziel in überprüfbare Schritte zerlegt; jeder Schritt braucht ein erwartetes Ergebnis. Ein Re-Plan ist nötig bei Tool-Fehler, widersprüchlichen Daten oder verändertem Ziel — Änderungen am Ziel oder an Berechtigungen brauchen neue Freigabe.

Freigabeschritte

  • Freigaben an konkrete Risiken binden (Empfänger, Betrag, Datensätze, Änderung).
  • Freigabe vor dem Tool-Aufruf, im Audit-Log nachvollziehbar.
  • Lesewerkzeuge standardmäßig erlauben, Schreibwerkzeuge standardmäßig sperren.
  • Erlaubte Ziele, Betragsgrenzen und Datenbereiche technisch beschränken.
  • Aktionen idempotent machen — Timeout darf keine Doppelbuchung erzeugen.
  • Hohe Risiken: Vier-Augen-Prinzip.

Fehlerbehandlung und Abbruch

Abbruchbedingungen sind zum Beispiel:

  • Zeit-, Token- oder Kostenbudget überschritten.
  • Maximale Anzahl von Tool-Aufrufen erreicht.
  • Berechtigung fehlt oder Datenquelle nicht erreichbar.
  • Ergebnis widerspricht einer fachlichen Regel.
  • Notwendige Nutzerfreigabe bleibt aus.
  • Anzeichen für Prompt Injection oder unerwartete Tool-Eingabe.

Ein Abbruch mit verständlicher Begründung ist ein Qualitätsmerkmal, kein Versagen.

Take-aways

  • Nicht jeder Prozess braucht einen Agenten — eine Pipeline ist oft die bessere Wahl.
  • Autonomie ist eine Prozessentscheidung, keine Modell-Eigenschaft.
  • Werkzeuge klein, typisiert, reversibel — Schreibwerkzeuge default sperren.
  • Abbruch ist ein Feature, kein Versagen — solange er begründet ist.
