Prompt Engineering

Phase 1 — Aus einem schlechten Prompt einen stabilen Business-Prompt machen

Lernziel

  • Den Aufbau eines guten Prompts aus wenigen Bausteinen zusammensetzen.
  • Zero-Shot, Few-Shot und iteratives Prompting unterscheiden.
  • Bekannte Patterns und typische Fehler kennen.
  • Verstehen, warum Prompt Engineering allein nicht reicht.

Aufbau eines guten Prompts

ElementZweck
RolleWer soll das Modell sein?
ZielWas soll erreicht werden?
KontextWorum geht es, was ist Hintergrund?
AufgabeKonkrete Anweisung
EinschränkungenWas darf nicht?
AusgabeformatWie soll das Ergebnis aussehen?
BeispieleFew-Shot-Vorgaben

Beispiel-Skelett

Rolle:  Du bist Senior-Risikoanalyst in einer Bank.
Ziel:   Erkenne vertragliche Risiken in Vertragstexten.
Kontext: Wir prüfen deutsche Lieferverträge im B2B-Bereich.
Aufgabe: Liefere je Klausel eine Risikoeinschätzung (hoch/mittel/niedrig).
Einschränkungen: Keine Rechtsberatung; immer Quelle aus dem Text zitieren.
Format:  Tabelle mit Spalten: Klausel | Risiko | Begründung | Quelle.
Beispiele: Siehe Anhang A.

Zero-Shot vs. Few-Shot

  • Zero-Shot: nur die Aufgabe, keine Beispiele
  • Few-Shot: 1–5 Beispiele für das gewünschte Schema

Few-Shot ist robuster für strukturierte Aufgaben, kostet aber Tokens.

Iteratives Prompting

Ein guter Prompt entsteht fast nie im ersten Versuch.

  1. Ersten Wurf schreiben
  2. An 5–10 Beispielen ausführen
  3. Schwächen identifizieren (Halluzinationen, falsches Format, fehlender Kontext)
  4. Prompt präzise anpassen
  5. Wiederholen, bis Stabilität erreicht ist

Prompt Templates & Versionierung

  • Wiederkehrende Aufgaben: Prompt als Vorlage mit Platzhaltern
  • Versionierung in Git — Änderungen nachvollziehbar machen
  • Tools: promptr, LangChain-Templates oder einfach Markdown-Dateien

Tipp: Prompt + Beispiel-Set gehören zusammen versioniert.

Rückfragen erlauben

Modelle dürfen (und sollen) Rückfragen stellen, wenn die Aufgabe mehrdeutig ist. Das senkt die Halluzinationsrate drastisch.

Wenn etwas unklar ist, frage nach, statt zu raten.

Typische Fehler

  • Mehrere Aufgaben in einem Prompt vermischen
  • Implizite Annahmen statt explizitem Kontext
  • Fehlende Formatvorgabe — das Modell „rät"
  • Negative Anweisungen („sag nicht X") statt positiver Formulierung
  • Zu lange Prompts mit redundantem Kontext

Prompt Engineering allein reicht nicht

Für eine zuverlässige AI-Anwendung braucht es zusätzlich:

  • Evaluation mit qualifizierten Testfällen
  • Vordefinierte Tools / Function Calling
  • Quellen (RAG) statt freier Erfindung
  • Human-in-the-Loop bei Konsequenzen

Prompt Engineering ist notwendig, aber nicht hinreichend.
