Was sind LLMs?

Large Language Models sind Modelle, die auf riesigen Textmengen trainiert wurden und Wahrscheinlichkeiten für das nächste wahrscheinlichste Token berechnen.

Aus dieser einfachen Aufgabe entsteht durch Skalierung erstaunlich vielseitige Intelligenz.

Tokens & Kontext

  • Token: Wortteile; ca. 0,5–1 Token pro englisches Wort, Deutsch 1,2–2 Tokens pro Wort
  • Kontextfenster: maximale Länge aus Eingabe + Ausgabe, die das Modell gleichzeitig verarbeiten kann
  • Was nicht ins Fenster passt, „weiß" das Modell nicht.

Stand: 23.07.2026 — siehe Übersicht auf der „Anbieter im Vergleich"-Folie.

Temperatur und Determinismus

  • T = 0: greedy decoding, sehr deterministisch
  • T ≈ 0,7: ausgewogen (typisch)
  • T ≥ 1: sehr kreativ, höhere Halluzinationsrate

Für reproduzierbare Ausgaben: Temperatur 0, ggf. seed-Parameter festlegen.

System-Prompt, User-Prompt, Assistant

  • System-Prompt: Rollen- und Verhaltensanweisungen (von Entwicklern)
  • User-Prompt: die eigentliche Anfrage
  • Assistant-Response: die Antwort des Modells

Konvention: System = Rolle & Regeln, User = Frage, Assistant = Antwort.

Halluzinationen

Eine Halluzination ist eine falsche Aussage, die das Modell sicher präsentiert.

  • Trainingsdaten enthalten den Fakt nicht
  • Verteilung erlaubt plausible Lügen
  • Modell unterscheidet nicht zwischen „wird oft gesagt" und „ist wahr"

Erkennung: Quellen verlangen, Cross-Check, gesundes Misstrauen.

Strukturierte Ausgaben & Multimodalität

  • Strukturierte Ausgaben: response_format / JSON Mode — zwingt das Modell in valide JSON- oder XML-Strukturen
  • Multimodale Modelle: verarbeiten Text, Bilder, Audio und teilweise Video
  • Use Cases: Rechnungsscan, Diagrammanalyse, Audio-Transkription

Reasoning-Modelle

Modelle wie OpenAI o3 / o4 oder Claude mit „Extended Thinking" zerlegen Aufgaben intern in Zwischenschritte.

  • Langsamer und teurer pro Token
  • Oft deutlich bessere Ergebnisse bei Mathematik, Logik und mehrstufiger Planung
  • Trade-off: Qualität vs. Latenz / Kosten

Grenzen von LLMs

  • Fehlendes aktuelles Wissen — Trainingsstand beachten
  • Falsche Quellenangaben — erfundene Paper, Urteile, URLs
  • Rechenfehler — selbst starke Modelle rechnen nicht zuverlässig
  • Scheinsicherheit — das Modell sagt selten von sich aus „ich weiß es nicht"
  • Kontextverlust — ältere Teile langer Konversationen werden unscharf

Praxisregel: „Das klingt überzeugend" ist das schwächste Argument für Korrektheit.
